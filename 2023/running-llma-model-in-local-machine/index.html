Running LLaMA model locally

April 30, 2023 - 2 minutes read - 344 words

Preparation

(30 mins)

LLaMA is a collection of foundation language models ranging from 7B to 65B parameters.

In this guide, I will be using and following Georgi Gergano’s llama.cpp, a inference of LLaMA model in pure C/C++.

I will be setting this up in a Ubuntu machine with 32Gb.

To prepare for the build system, I installed these:

sudo apt install git cmake build-essential python3 python3-pip
  1. Clone and build the C/C++ codes:
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp

# using CMake
mkdir build
cd build
cmake ..


cmake --build . --config Release

  1. For the 7B model (which runs faster as compared to the 13B model), I downloaded the ggml format via here:
wget https://huggingface.co/Pi3141/alpaca-7B-ggml/resolve/main/ggml-model-q4_0.bin

For the 13B model, I downloaed from here:

wget https://huggingface.co/Pi3141/gpt4-x-alpaca-native-13B-ggml/resolve/main/ggml-model-q4_0.bin
  1. That’s all! By following the provided setup, you can now comfortably run LLaMA (ChatGPT-like) model on your local machine without any worries about exposing your prompt or data.

Running LLaMA model locally

(5 mins)

7B Model

For the 7B model (under models/7B folder), using the pre-trained GPT model by:

 ./build/bin/main -m ./models/7B/ggml-model-q4_0.bin -n 128 --repeat_penalty 1.0 --color -i -r "User:" -f prompts/chat-with-bob.txt

This is the sample response for my prompt:

how to create a chatgpt like app in flutter, give me the step by step code starting with the main.dart

13B Model

For the 13B model (under models/13B folder), using the pre-trained GPT model by:

./build/bin/main -m ./models/13B/ggml-model-q4_0.bin -n 256 --repeat_penalty 1.0 --color -i -r "User:" -f prompts/chat-with-bob.txt

Running with OpenBLAS (optional)

(2 mins)

OpenBLAS is an optimized Basic Linear Algebra Subprograms (BLAS) library. You may install with:

sudo apt-get install libopenblas-dev

Rebuilding llama with OpenBLAS on,

cmake .. -DLLAMA_OPENBLAS=ON
cmake --build . --config Release

NOTE: For the latest version, to work with OpenBLAS, here’s the command. You may refer to the this) for more information of how to run with ggjt v3:

cmake .. -DLLAMA_BLAS=ON -DLLAMA_BLAS_VENDOR=OpenBLAS
cmake --build . --config Release
